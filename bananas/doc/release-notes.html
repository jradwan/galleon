Welcome to Bananas

The HME home page is hosted on sourceforge, http://tivohme.sourceforge.net, and contains news, updates, related downloads and support resources. Please do not call TiVo customer service as they will be unable to help you with the HME SDK or the Bananas UI Toolkit.

Bananas is a TiVo-style user interface toolkit built on top of the HME SDK. Bananas makes it easy to create great looking applications that are compatible with the TiVo DVR look-and-feel. You should be familiar with HME before attempting to build applications using Bananas.

Requirements

In order to use the TiVo HME SDK, you should:
  • Own at least one of the following, with TiVo Service : a TiVo Series2 DVR or a Toshiba DVD Player (no combination TiVo/DIRECTV DVRs)
  • Be running TiVo service update 7.2-X or higher. Your box will automatically receive this release through a scheduled rollout process.
  • Have a network with your TiVo on it, wired or wireless
  • Have Windows XP, Linux, or Mac OS X
  • Be proficient at developing in Java
  • Run JDK 1.4.1 or higher

Getting Started

First, on your PC:
  1. Set the HME_HOME environment variable to point at the HME SDK. For example: 

    > export HME_HOME=~/hme_sdk_1.3   (Linux/Mac OS)
    > set HME_HOME=c:\hme_sdk_1.3     (Windows)

  2. Run the Bananas sample: 

    > cd samples
    > runsample.sh   (use runsample.bat on Windows)

Recent Changes

08/12/2005 - 1.3 Release

  • New Bananas Developer's Guide - Please refer to this guide for detailed information on the Bananas UI Toolkit.
  • New BKeyboard object.
  • New BSkin subclasses to make working with custom skins easier.
  • The replaceKeyboard() method now correctly looks for the longest row.
  • Added null check to dispatchKeyEvent() to ensure a screen exists before dispatching key events. 
  • System now plays "bonk"' if no screen is available.
  • Fixed private/public bug in BSkin.Element.
  • Added getTop() and getNVisibleRows() to BList, as well as a setView() to BHighlight.
  • Keyboard generates a Keyboard event, which is a subclass of HmeEvent rather than an Action event. This is to eliminate clashes with current action events if user types a word that is a pre-defined action (e.g., "up" or "left")
  • Updated BKeyboard to use post event instead of IKeyboardClient; use Keyboard class instead of large 3 dimensional arrays; when keyboard changes, select item in same space rather than item with same name.
  • Updated bananas sample to include keyboard sample.
